# LeetCode 200 - 岛屿数量

## 题目概述

给定一个由 `'1'`（陆地）和 `'0'`（水）组成的二维网格，计算网格中岛屿的数量。岛屿由水平或垂直方向相邻的陆地连接而成，被水包围。

## 解题思路

这道题的核心是**连通分量检测**——在网格图中找出所有相互连通的"1"组成的区域。提供三种经典解法：

### 解法一：DFS 沉岛法

**核心思想**：遍历网格，遇到 `'1'` 就启动 DFS，把与之相连的所有陆地"沉掉"（标记为 `'0'`），每启动一次 DFS 就代表发现一个新岛屿。

1. 遍历网格，遇到 `'1'` → 岛屿计数+1
2. 从该点出发 DFS，将连通的所有 `'1'` 改为 `'0'`
3. 继续遍历，已被沉掉的岛不会再被计数

**优点**：代码最简洁，原地修改不需要额外空间（递归栈除外）

**缺点**：递归深度在大网格下可能栈溢出（LeetCode 上通常 OK，但实际工程中需要注意）

### 解法二：BFS 迭代法

**核心思想**：与 DFS 相同的沉岛逻辑，但用队列实现迭代版本。

1. 遍历网格，遇到 `'1'` → 岛屿计数+1
2. 将该点入队，BFS 扩展四方向邻居
3. 访问到的陆地标记为 `'0'`

**优点**：无栈溢出风险，适合大规模网格

**缺点**：队列的额外空间开销

### 解法三：并查集（Union-Find）

**核心思想**：将每个陆地格子视作节点，相邻陆地合并到同一集合，最终统计集合数量。

1. 初始化：每个格子独立为一个集合，统计陆地总数
2. 遍历网格，只向右和向下检查（避免重复）
3. 相邻陆地若属于不同集合 → 合并，陆地计数-1
4. 最终陆地计数即为岛屿数量

**优点**：适合动态场景（频繁合并/查询），路径压缩+按秩合并让操作接近 O(1)

**缺点**：代码量较大，初始化需要额外 O(m×n) 空间

## 复杂度分析

| 解法 | 时间复杂度 | 空间复杂度 | 说明 |
|------|-----------|-----------|------|
| **DFS** | O(m × n) | O(m × n) 最坏 | 每格访问一次；空间为递归栈深度，最坏整个网格都是陆地 |
| **BFS** | O(m × n) | O(min(m, n)) | 队列最大宽度为网格较短边的规模 |
| **Union-Find** | O(m × n × α(mn)) | O(m × n) | α 为反阿克曼函数，实际可视为 O(1) |

> **注意**：BFS 的空间复杂度 O(min(m,n)) 是一个经典结论——在网格 BFS 中，队列最大长度受限于对角线方向的宽度，约为 min(m,n)。

| 场景 | 对应概念 | 具体应用 |
|------|---------|---------|
| **领土/势力系统** | 连通分量 | 检测玩家控制的区域是否连通，割裂则分裂为多个势力 |
| **程序化地形** | Flood Fill | 生成地图后标记连通区域，确保关卡可通行 |
| **战争迷雾** | 区域扩散 | 已探索区域的连通性判断，迷雾解锁范围 |

| **AI 寻路前置** | 可达性检测 | 判断起点终点是否在同一连通分量，避免无效寻路 |

**UE5 实际应用**：
- `UNavigationSystemV1` 中 NavMesh 的连通区域检测，本质上就是岛屿数量问题的工程化版本
- 开放世界中的领地系统（如 Rust/方舟），需要实时检测建筑群的连通性
- 程序化关卡生成后，用 Flood Fill 验证地图可通关性

## 优化方向

### 1. 避免原地修改
如果需要保留原始网格，用 `visited` 数组替代沉岛法：
```cpp
vector<vector<bool>> visited(rows, vector<bool>(cols, false));
```
空间 O(m×n)，但不破坏输入数据。

### 2. 并查集的动态扩展
实际游戏中地图可能动态变化（玩家放置/移除方块），并查集支持：
- **增量合并**：新放置的方块与邻居合并，O(α(n))
- **分裂处理**：移除方块后需重新计算连通性（较复杂，可考虑延迟重建）

### 3. 大地图分块处理
对于超大规模网格（如开放世界），可以：
- 将地图分块（Chunk），每块独立计算连通性
- 跨块边界用并查集合并
- 这正是 UE5 NavMesh 分块构建的思路

## 变种题目

1. **LeetCode 695 - 岛屿的最大面积**：求最大连通分量的大小，DFS 时计数即可
2. **LeetCode 463 - 岛屿的周长**：遍历每块陆地，检查四方向是否为水或边界
3. **LeetCode 130 - 被围绕的区域**：先从边界 Flood Fill 标记不被包围的 `'O'`
4. **LeetCode 1254 - 统计封闭岛屿数目**：边界连通的不算封闭岛屿

## 三种解法选择指南

```
需要代码简洁？          → DFS 沉岛法
担心栈溢出？            → BFS 迭代法
需要动态增删/频繁查询？  → Union-Find
```

## 总结

岛屿数量是**网格图连通分量**的经典模板题，三种解法各有适用场景：
- DFS 最简洁，面试首选
- BFS 最安全，工程首选
- Union-Find 最灵活，动态场景首选

这道题在游戏开发中应用极广，从地图生成验证到领地系统都有它的影子，务必熟练掌握！
