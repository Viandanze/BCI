# LeetCode 210 - 课程表 II (Course Schedule II)

## 题目概述

现有 `numCourses` 门课需要选，记为 `0` 到 `numCourses-1`。给定 `prerequisites` 数组，其中 `prerequisites[i] = [ai, bi]` 表示选修课程 `ai` 前必须先选修课程 `bi`。返回完成所有课程的学习顺序，如果不可能完成则返回空数组。

**本质**：有向图的拓扑排序问题。如果图中有环则无解，否则输出一个合法的拓扑序列。

## 解题思路

### 解法一：BFS 拓扑排序（Kahn 算法）—— 面试首选

**核心思想**：不断移除入度为 0 的节点（没有前置依赖的课），直到所有节点被移除或无法继续。

1. 构建邻接表 + 入度数组
2. 所有入度为 0 的节点入队
3. 每次出队一个节点，将其所有后继节点的入度 -1
4. 入度减到 0 的节点入队
5. 若最终排序结果包含全部节点 → 成功；否则存在环 → 返回空数组

**优点**：代码清晰，逻辑直观，天然按"层级"输出（同层节点可并行），适合游戏中的并发加载场景

**缺点**：需要额外空间存入度数组和队列

### 解法二：DFS 拓扑排序 —— 逆后序即拓扑序

**核心思想**：对图做 DFS，节点的完成顺序（后序）的逆序就是拓扑序。同时用三色标记法检测环。

1. 状态数组：0=未访问，1=访问中（当前路径），2=已完成
2. DFS 过程中遇到状态为 1 的节点 → 存在环（反向边）
3. 节点所有邻居处理完毕后，记录到结果数组（后序位置）
4. 最终将结果数组逆序即为拓扑排序

**为什么逆后序是拓扑序？**
- 后序遍历中，被依赖的节点（前置课）会后于依赖它的节点被记录
- 逆序后，前置课就排到了前面

**优点**：一次遍历同时完成拓扑排序和环检测，代码紧凑

**缺点**：递归版本可能栈溢出；不如 BFS 直观

### 解法三：DFS 迭代版 —— 避免栈溢出

**核心思想**：手动维护栈模拟递归，逻辑与递归版完全一致，但不受调用栈深度限制。

1. 用 `(节点, 邻居迭代索引)` 的栈模拟递归
2. 入栈标记为 1，出栈标记为 2 并记录后序
3. 遇到状态 1 的节点 → 检测到环

**优点**：无栈溢出风险，适合大规模图

**缺点**：代码较复杂，迭代模拟递归不如直接递归直观

## 复杂度分析

| 解法 | 时间复杂度 | 空间复杂度 | 说明 |
|------|-----------|-----------|------|
| **BFS (Kahn)** | O(V + E) | O(V + E) | 邻接表 O(V+E)，入度数组 O(V)，队列 O(V) |
| **DFS 递归** | O(V + E) | O(V + E) | 邻接表 O(V+E)，状态数组 O(V)，递归栈 O(V) |
| **DFS 迭代** | O(V + E) | O(V + E) | 与递归版相同，手动栈替代递归栈 |

> **V = numCourses, E = prerequisites.size()**

三种解法的时间复杂度相同，都是标准的图遍历复杂度 O(V+E)。区别在于常数因子和工程适用性。

## 优化方向

### 1. 字典序最小的拓扑排序

当需要"确定性"的排序结果时（如多人同步游戏中的加载顺序必须一致），可用优先队列替代普通队列：

```cpp
// BFS 中用优先队列替代普通队列
priority_queue<int, vector<int>, greater<int>> pq;  // 小顶堆
```

时间复杂度变为 O(V log V + E)，但保证了唯一性和确定性。

### 2. 动态增删依赖

实际游戏中任务依赖可能动态变化（如版本更新新增任务线），需要支持：
- **增量拓扑排序**：新依赖加入时局部更新，而非全局重排
- **环的快速定位**：检测到环时，返回具体是哪些节点形成的环（调试用）

### 3. 并行执行优化

BFS 拓扑排序的"层级"特性天然适合并行：同一轮出队的节点互不依赖，可以同时执行。在游戏资源加载场景中，这意味着同层资源可以并行加载，显著减少总加载时间。

```
Level 0: [A]           ← 先加载
Level 1: [B, C]        ← 可并行
Level 2: [D]           ← 再加载
```

### 4. 关键路径分析

对任务图做最长路径分析（加权的拓扑排序），可以找出"关键路径"——决定最短完成时间的瓶颈链路。在项目管理（UE5 项目里程碑规划）和游戏关卡设计中有直接应用。

## 变种题目

1. **LeetCode 207 - 课程表**：只判断能否完成（拓扑排序/环检测），不要求输出顺序，是本题的简化版
2. **LeetCode 269 - 火星词典**：根据排序规则推断字母顺序，本质也是拓扑排序
3. **LeetCode 329 - 矩阵中的最长递增路径**：拓扑排序 + DP，按层级递推最长路径
4. **LeetCode 1136 - 并行课程**：求最少学期数完成所有课程，就是 BFS 层级数的概念

## 三种解法选择指南

```
面试快速写？         → BFS (Kahn)，逻辑最直观
需要检测环？         → DFS 三色标记，一次遍历同时完成
图规模大怕栈溢出？   → DFS 迭代版
需要并行/层级信息？  → BFS，天然按层输出
```

## 总结

课程表 II 是**拓扑排序**的经典模板题，三种解法各有适用场景：
- **BFS (Kahn)**：面试首选，逻辑清晰，天然支持层级/并行
- **DFS 递归**：代码紧凑，逆后序的理解是核心
- **DFS 迭代**：工程首选，无栈溢出风险

拓扑排序在游戏开发中无处不在——从技能树到任务链到资源加载，任何有"先后依赖"关系的系统都是它的应用场景。这道题务必熟练掌握，面试中拓扑排序是高频考点！
