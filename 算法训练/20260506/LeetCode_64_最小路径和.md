# LeetCode 64 - 最小路径和

## 题目描述

给定一个 `m × n` 的网格 `grid`，网格中的每个单元格包含一个非负费用。找到一条从左上角到右下角的路径，使得路径上所有单元格费用之和最小。每次只能向下或向右移动。

### 游戏开发场景

- 🎮 **角色寻路**：游戏中角色从起点到终点的最短路径规划
- 🚀 **RTS单位移动**：实时战略游戏中单位的移动路径选择
- 🗺️ **自动寻路系统**：A*算法的简化基础版本
- 🏰 **地牢探索**：最优路线计算

## 解题思路

### 核心思想：动态规划

**状态定义**：`dp[i][j]` = 从起点 `(0,0)` 到位置 `(i,j)` 的最小路径和

**状态转移方程**：
```
dp[i][j] = grid[i][j] + min(dp[i-1][j], dp[i][j-1])
```

**边界条件**：
- `dp[0][0] = grid[0][0]`（起点）
- 第一行：`dp[0][j] = dp[0][j-1] + grid[0][j]`
- 第一列：`dp[i][0] = dp[i-1][0] + grid[i][0]`

### 为什么可以用DP？

1. **最优子结构**：从 `(0,0)` 到 `(i,j)` 的最短路径，取决于到 `(i-1,j)` 或 `(i,j-1)` 的最短路径
2. **无后效性**：每个位置的解只依赖前面位置的解，不受后续选择影响
3. **重叠子问题**：递归公式结构清晰，存在大量重复计算

## 复杂度分析

### Solution 1：基础动态规划

| 指标 | 复杂度 | 说明 |
|------|--------|------|
| 时间 | O(m × n) | 遍历整个网格一次 |
| 空间 | O(m × n) | 需要完整的dp表 |

### Solution 2：空间优化

| 指标 | 复杂度 | 说明 |
|------|--------|------|
| 时间 | O(m × n) | 仍需遍历整个网格 |
| 空间 | O(n) | 只保存一行数据 |

**优化原理**：
- `dp[j]` 更新前 = `dp[i-1][j]`（上一行的值）
- `dp[j-1]` 更新后 = `dp[i][j-1]`（当前行左边的值）
- 通过从左到右遍历，利用已更新的 `dp[j-1]`

### Solution 3：原地修改

| 指标 | 复杂度 | 说明 |
|------|--------|------|
| 时间 | O(m × n) | 遍历整个网格 |
| 空间 | O(1) | 直接修改原网格 |

## 游戏开发扩展

### 扩展到A*寻路算法

这道题是A*寻路的基础，两者核心思想相同：

| 特性 | 本题 | A*算法 |
|------|------|--------|
| 移动限制 | 只能下/右 | 八方向/六方向 |
| 启发函数 | 无 | Manhattan/Euclidean |
| 代价计算 | 格子代价 | G(n) + H(n) |

### 变种题目推荐

1. **LeetCode 62 - 不同路径**：不计代价，只计路径数
2. **LeetCode 63 - 不同路径 II**：带障碍物的路径计数
3. **LeetCode 174 - 地下城游戏**：从左上到右下的最小生命值
4. **LeetCode 1301 - 最大得分的路径数目**：带奖励的路径选择

## 优化方向

1. **对角线优化**：当 m ≈ n 时，可考虑矩阵对角线遍历
2. **缓存友好**：按行遍历比按列遍历缓存命中率高
3. **SIMD加速**：大量整数加法和比较可用SIMD并行化
