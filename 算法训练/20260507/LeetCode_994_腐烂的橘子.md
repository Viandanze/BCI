# LeetCode 994 - 腐烂的橘子

## 题目概述

在网格中模拟腐烂橘子扩散，求全部橘子腐烂的最少分钟数。

## 解题思路

### 核心算法：多源 BFS（广度优先搜索）

1. **初始化**：遍历网格，将所有腐烂橘子(值为2)加入队列，统计新鲜橘子数量
2. **边界处理**：如果没有新鲜橘子，直接返回0
3. **BFS 扩散**：
   - 从队列取出腐烂橘子
   - 向四个方向扩散（上右下左）
   - 如果相邻是新鲜橘子，腐烂它（标记为2），并加入队列，时间+1
   - 更新最大时间
4. **结果判断**：BFS结束后，检查是否还有未腐烂的新鲜橘子

### 游戏开发应用场景

| 场景 | 腐烂橘子 | 新鲜橘子 | 扩散效果 |
|------|----------|----------|----------|
| 火焰蔓延 | 火源 | 可燃物 | 火烧蔓延 |
| 病毒感染 | 感染者 | 健康单位 | 疾病传播 |
| 毒圈系统 | 毒区边界 | 安全区 | 毒圈收缩 |
| 范围伤害 | 爆炸中心 | 附近敌人 | AOE伤害 |

## 代码实现

```cpp
int orangesRotting(vector<vector<int>>& grid) {
    queue<Point> q;
    int freshCount = 0;
    
    // 多源初始化
    for (int i = 0; i < rows; ++i)
        for (int j = 0; j < cols; ++j)
            if (grid[i][j] == 2) q.push({i, j, 0});
            else if (grid[i][j] == 1) freshCount++;
    
    if (freshCount == 0) return 0;
    
    // BFS 扩散
    while (!q.empty()) {
        Point curr = q.front(); q.pop();
        for (int dir = 0; dir < 4; ++dir) {
            int nx = curr.x + dx[dir];
            int ny = curr.y + dy[dir];
            if (valid && grid[nx][ny] == 1) {
                grid[nx][ny] = 2;
                freshCount--;
                q.push({nx, ny, curr.time + 1});
            }
        }
    }
    
    return freshCount == 0 ? maxTime : -1;
}
```

## 复杂度分析

| 复杂度 | 分析 | 结果 |
|--------|------|------|
| **时间复杂度** | 每个格子最多被访问一次（入队一次） | **O(m × n)** |
| **空间复杂度** | 队列最坏情况下存储所有腐烂橘子 | **O(m × n)** |

## 优化方向

### 1. 空间优化
- 可以用 `deque` 代替 `queue`，手动维护层的边界
- 或者在网格中直接记录时间（原位修改）

### 2. 提前终止
```cpp
// 如果当前分钟已经感染了所有橘子，可以提前返回
if (freshCount == 0) return curr.time;
```

### 3. 并行优化（GPU场景）
- 大规模网格可用 CUDA/OpenCL 并行化
- 每个腐烂源独立计算扩散，相互独立

## 总结

这道题是**多源BFS**的经典应用，核心思想是：
- 从多个起点同时开始BFS
- 层级遍历记录扩散时间
- 判断是否所有目标都被覆盖

这种模式在游戏开发中非常常见，建议熟练掌握！
