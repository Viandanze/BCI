## 核心概念速览

### 为什么需要边缘计算？

```
┌─────────────────────────────────────────────────────────────────┐
│                    数字孪生系统响应时间对比                        │
├─────────────────────────────────────────────────────────────────┤
│  纯云端方案                                                     │
│  传感器 → 网络传输 → 云端处理 → 响应                            │
│  延迟：200-500ms（不适合实时控制）                               │
├─────────────────────────────────────────────────────────────────┤
│  云边协同方案（推荐）                                            │
│  边缘侧：数据采集 → 本地处理 → 毫秒响应                          │
│  云端侧：历史分析 → 模型训练 → 策略下发                          │
│  延迟：边缘 10-50ms，云端负责复杂分析                           │
└─────────────────────────────────────────────────────────────────┘
```

### 边缘计算三大价值

| 价值 | 说明 | 你的场景 |
|------|------|---------|
| **低延迟** | 本地处理，毫秒级响应 | 传感器异常实时告警 |
| **带宽节省** | 80%数据在边缘清洗，仅上传关键指标 | 减少MQTT消息量 |
| **断网自治** | 本地缓存决策，网络恢复后同步 | 树莓派离线运行 |

### 任务2：一键启动边缘服务

创建 `~/digital-twin/docker-compose.yml`：

```yaml
version: "3.8"

services:
  # MQTT Broker（消息中枢）
  mqtt:
    image: eclipse-mosquitto:2.0
    container_name: dt-mqtt
    ports:
      - "1883:1883"   # MQTT端口
      - "9001:9001"   # WebSocket端口
    volumes:
      - ./mosquitto/config:/mosquitto/config
      - ./mosquitto/data:/mosquitto/data
      - ./mosquitto/log:/mosquitto/log
    restart: unless-stopped

  # 时序数据库（数据存储）
  influxdb:
    image: influxdb:2.7
    container_name: dt-influx
    ports:
      - "8086:8086"
    volumes:
      - ./influxdb:/var/lib/influxdb2
    restart: unless-stopped

  # 可视化仪表盘
  grafana:
    image: grafana/grafana:10
    container_name: dt-grafana
    ports:
      - "3000:3000"
    volumes:
      - ./grafana:/var/lib/grafana
    depends_on:
      - influxdb
    restart: unless-stopped

  # 边缘流处理（Node-RED）
  nodered:
    image: nodered/node-red:3.1
    container_name: dt-nodered
    ports:
      - "1880:1880"
    volumes:
      - ./nodered:/data
    depends_on:
      - mqtt
    restart: unless-stopped
```

创建 `~/digital-twin/mosquitto/config/mosquitto.conf`：

```conf
# 允许匿名访问（仅限本地开发）
allow_anonymous true

# 持久化配置
persistence true
persistence_location /mosquitto/data/

# 日志配置
log_dest file /mosquitto/log/mosquitto.log

# 监听端口
listener 1883
listener 9001
protocol websockets
```

### 任务3：启动并验证服务

```bash
# 创建目录
```

## 实战任务：数据采集到可视化闭环

### 完整数据流设计

```
Arduino传感器 → 串口 → Python采集程序 → MQTT → 
  ├→ Node-RED（边缘处理）→ InfluxDB（存储）
  │                            ↓
  │                      Grafana（仪表盘）
  │                            ↓
  └→ Web浏览器 → Three.js（3D可视化）
```

### 采集程序示例（Python）

```python
#!/usr/bin/env python3
"""
数字孪生传感器采集程序
将Arduino数据通过MQTT发送到边缘网关
"""

import serial
import paho.mqtt.client as mqtt
import json
import time

# 配置
SERIAL_PORT = "/dev/ttyUSB0"  # 根据实际调整
BAUD_RATE = 115200
MQTT_BROKER = "localhost"
MQTT_PORT = 1883
MQTT_TOPIC = "sensors/dht11"

def on_connect(client, userdata, flags, rc):
    if rc == 0:
        print(f"✅ MQTT连接成功: {MQTT_BROKER}:{MQTT_PORT}")
    else:
        print(f"❌ MQTT连接失败，错误码: {rc}")

def read_serial():
    """从Arduino读取传感器数据"""
    try:
        ser = serial.Serial(SERIAL_PORT, BAUD_RATE, timeout=1)
        line = ser.readline().decode('utf-8').strip()
        # 假设Arduino输出格式: {"temp":25.5,"hum":60.2}
        data = json.loads(line)
        return data
    except Exception as e:
        print(f"⚠️ 串口读取异常: {e}")
        return None

def main():
    # MQTT客户端
    client = mqtt.Client()
    client.on_connect = on_connect
    client.connect(MQTT_BROKER, MQTT_PORT, 60)
    client.loop_start()
    
    print("🚀 传感器采集程序启动")
    print(f"   串口: {SERIAL_PORT}")
    print(f"   MQTT: {MQTT_BROKER}:{MQTT_PORT}")
    
    while True:
        data = read_serial()
        if data:
            # 发布到MQTT
            client.publish(MQTT_TOPIC, json.dumps(data))
            print(f"📤 已发送: {data}")
        time.sleep(2)  # 2秒采集一次

if __name__ == "__main__":
    main()
```
