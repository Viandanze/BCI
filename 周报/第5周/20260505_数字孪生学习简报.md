## 核心概念速览

### OPC UA vs MQTT 选型决策树

```
需要语义建模+复杂关系？
    ├─ YES → OPC UA（工业现场级）
    └─ NO ↓
需要云端轻量传输？
    ├─ YES → MQTT（云边协同）
    └─ NO → 仅用本地存储

主流融合架构（必考）：
[传感器/PLC] → [OPC UA边缘服务器] → [MQTT桥接器] → [云端IoT平台]
```

### OPC UA三大核心能力

| 能力 | 说明 | 工业价值 |
|------|------|----------|
| **地址空间** | 层次化节点树，携带完整元数据 | 语义自描述，无需额外文档 |
| **Pub/Sub扩展** | 支持MQTT/AMQP等传输映射 | 打通IT/OT鸿沟 |
| **安全机制** | X.509证书 + AES加密 + 用户认证 | 企业级安全标准 |

### python-opcua 生态选择

| 库 | 特点 | 适用场景 |
|----|------|----------|
| **python-opcua** | 同步API，纯Python，v1.1.1（2026.1） | 树莓派/边缘网关/快速验证 |
| **opcua-asyncio** | 异步API，同一团队 | 高并发(50+连接)或FastAPI混用 |

**结论**：你的树莓派5用 `pip install opcua` 即可，够用。

### OPC UA服务器架构（树莓派5部署）

```python
# opcua_server.py - 边缘网关OPC UA服务器
from opcua import Server
import json

# 1. 创建服务器
server = Server()
server.set_endpoint("opc.tcp://0.0.0.0:4840/freeopcua/server/")

# 2. 注册命名空间（关键！漏了客户端找不到节点）
uri = "http://digital-twin.raspberry-pi"
idx = server.register_namespace(uri)

# 3. 创建对象和变量节点
objects = server.get_objects_node()
my_object = objects.add_object(idx, "SensorGateway")

# 温度变量（携带元数据）
temp_var = my_object.add_variable(idx, "Temperature", 0.0)
temp_var.set_writable(True)  # 允许写入

# 湿度变量
humid_var = my_object.add_variable(idx, "Humidity", 0.0)

# 4. 启动服务器
server.start()
print("OPC UA Server started on opc.tcp://0.0.0.0:4840")

try:
    while True:
        # 模拟传感器更新
        import random
        temp_var.set_value(round(random.uniform(20, 30), 2))
        humid_var.set_value(round(random.uniform(40, 70), 2))
finally:
    server.stop()
```

### OPC UA客户端（消费数据）

```python
# opcua_client.py - 消费OPC UA数据
from opcua import Client

client = Client("opc.tcp://192.168.1.100:4840/freeopcua/server/")
try:
    client.connect()
    
    # 通过路径访问节点
    root = client.get_root_node()
    obj = root.get_child(["0:Objects", f"{idx}:SensorGateway"])
    temp = obj.get_child([f"{idx}:Temperature"])
    humid = obj.get_child([f"{idx}:Humidity"])
    
    print(f"Temperature: {temp.get_value()}°C")
    print(f"Humidity: {humid.get_value()}%")
    
finally:
    client.disconnect()
```

### 练习2：订阅模式（实时数据变更通知）

```python
# opcua_subscribe.py
from opcua import Client

def on_value_changed(node, val, data):
    print(f"Value changed: {val}")

client = Client("opc.tcp://192.168.1.100:4840")
client.connect()

# 创建订阅
sub = client.create_subscription(500, on_value_changed)  # 500ms采样

# 订阅节点
temp = client.get_node("ns=2;i=1")  # 根据实际NodeId
handle = sub.subscribe_data_change(temp)

input("Press Enter to exit...\n")
sub.unsubscribe(handle)
client.disconnect()
```

### 练习3：OPC UA → MQTT桥接（融合架构）

```python
# opcua_to_mqtt_bridge.py
from opcua import Client
import paho.mqtt.client as mqtt
import json

# 配置
OPCUA_URL = "opc.tcp://192.168.1.100:4840"
MQTT_BROKER = "localhost"
MQTT_TOPIC = "digital_twin/sensors"

# MQTT发布者
mqtt_client = mqtt.Client()
mqtt_client.connect(MQTT_BROKER)

# OPC UA客户端
opcua_client = Client(OPCUA_URL)

def on_notification(datachange_notif):
    for node in datachange_notif.monitored_items:
        payload = {
            "node_id": str(node.nodeid),
            "value": node.value.Value.Value,
            "timestamp": str(node.value.SourceTimestamp)
        }
        mqtt_client.publish(MQTT_TOPIC, json.dumps(payload))

async def main():
    opcua_client.connect()
    
    # 订阅多个节点
    subscription = opcua_client.create_subscription(1000, on_notification)
    
    nodes = [
        opcua_client.get_node("ns=2;i=1"),  # Temperature
        opcua_client.get_node("ns=2;i=2"),  # Humidity
        opcua_client.get_node("ns=2;i=3"),  # Pressure
    ]
    
    for node in nodes:
        subscription.subscribe_data_change(node)
    
    print("Bridge running...")
    
if __name__ == "__main__":
    import asyncio
    asyncio.run(main())
```
